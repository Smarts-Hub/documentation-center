If you wish to develop modules using TypeScript, we have made it easy for you by adding **types** to the api. <Badge text="From 1.1.0" variant="tip" />

:::caution
You might need to add your own `tsconfig.json`, since we are not adding it.
:::

## Using TypeScript types
To use types you shall import them

```ts
import type { BotModule, ModuleApi } from "0xbot";
```

To implement them into the `run` function you can follow this example:
```ts
export const run: BotModule["run"] = async (api: ModuleApi) => {
  // Your code goes here
};
```

## Using import aliases
With typescript you can still use import aliases just as normal:

```ts
import logger from "#logger";
```

### Available aliases
The available aliases are:
- `#logger` for `/core/logger.js`
- `#client` for `/core/bot/client.js`
- `#api` for `/core/modules/loader.js`

:::tip
Any of the above aliases can be imported from the `#api` alias. See [api](/0xbot/module-development/api/) for more information.
:::
